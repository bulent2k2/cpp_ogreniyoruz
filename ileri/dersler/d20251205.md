5. Dilim ağaçlarında acele edince işe şeytan karışıyor. Miskinlikte fayda var. Miskin olmayan haline dikkatle bakmıştık daha önce. [Atcoder sitesinden soru burada](https://atcoder.jp/contests/practice2/tasks/practice2_j). İki yönteme dikkat: 
```c++
// ...
using Dizi=std::vector<S>;
class DilimAğacı {
    // ...
public: 
    DilimAğacı(const Dizi& dizi);
    void değiştir(K konum; S yeniDeğer);
    Bilgi sor(Aralık aralık);
    // ...
};
```
Aralıklar hakkında soru sorup bilgi alabiliyoruz. Ama sadece tek bir elemanı değiştiriyoruz her seferinde. Kodun nasıl çalıştığına daha önce bakmıştık. 
Burda d bulunsun: [verimli bir çözüm örneği burada](https://onlinegdb.com/GoHYkq-uE).

Bu problemin biraz daha [genel hali de yine atcoder sitesinden burada](https://atcoder.jp/contests/practice2/tasks/practice2_l). Bu genelleme ve zorluk şu değişimden kaynaklanıyor:  
```c++
// ...
class DilimAğacı {
// ...
public: 
   // ...
   // bit dizisi için, aralıktaki birler sıfır, sıfırlar bir olsun:
   void değiştir(Aralık aralık);
   // ...
};
```
Yani, elemanları teker teker değiştirmek yerine bir aralıktaki elemanları hep birden değiştirmek istiyoruz. Bunu verimli yapmanın yolu nasıl olabilir? Gelecek derslerde göreceğiz inşallah. Ama, bu arada dilim ağacı yerine sadece basit bir dizi kullanarak çözmek iyi bir başlangıç olur. Verimi düşük, onun için testlerin hepsini beş saniye altında çözemez ama, aheste olmakta fayda var dedik ya. Onun için [başlangıç kodu burada](https://www.onlinegdb.com/dF2WKNR4e). Bir örnek çözüm de [burada](https://onlinegdb.com/ANDULgXji8) ama, bence kendiniz uğraşmadan önce hemen bakıvermeyin, acele etmeyin 🙂. Arkasından da aceleci dilim ağacı kodumuzu kullanarak çözün. Adım adım çözüme yaklaşmış ve bu ileri algoritmayı anlamaya hazırlanmış oluruz. O zaman da tam miskin çözüme bakarız.
