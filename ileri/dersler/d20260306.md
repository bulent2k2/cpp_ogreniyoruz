Canavarlardan Kaçış, Enlemesine Geziyle Arama ve Çalışma Hızı Karmaşıklığı
===

[Bir önceki dersin notlarına](d20260227.md) çabucak baktık. Dersimizin gerisinde [canavarlardan kaçış problemini](https://cses.fi/problemset/task/1194/) ve çözümünü derinlemesine inceledik. Takım arkadaşlarınız güzel sorular sordu. 

Sorulardan ikisi epey önemli:
1. enlemesine aramanın en kısa yolu bulduğunu nereden biliyoruz?
2. yavaş kodun yavaş çalışmasının nedeni nedir?

Problemi doğru çözen ama yeterince hızlı çalışmayan [kod burada](https://cses.fi/paste/cd4bda9bc7174903f9d7de/). 28 test arasından sadece 3 tanesi bir saniyeden çok zaman alıyor: 5, 6 ve 17 numaralı testler. Üçünün tahtası da 1000x1000 büyüklüğünde. Ama yeterince hızlı çalışan 19 numaralı test de 1000x1000 tahta kullanıyor. Yani hız farkının nedeni tahtanın büyüklüğü değil gibi görünüyor.

Yavaş kodla çok daha hızlı çalışan kodu ([o da burada](https://cses.fi/paste/53b00cbd34120c7ef9f8a2/)) karşılaştırdık. Yavaş program (141 satır) hızlı programdan (100 satır) epey daha uzun. İlginç, değil mi? Aynı programı tekrar tekrar yazdıkça daha az ve öz bir şekilde yazmak mümkün olabiliyor. 

Neyse, daha önemlisi hız farkının nedenine dönelim. Yavaş program önce şöyle yapıyor:
```c++
// her canavar için ayrı bir enlemesine gezi yap:
for(auto canavar: canavarlar) {
	canavarlara_uzaklık[canavar.y][canavar.x] = 0;
    kuyruk.push(canavar); 
	// ...
	while (not kuyruk.empty()) {
	   // tahta üzerindeki her noktanın bu canavara uzaklığını bul
	   // ...
    }
}
```
Hızlı program ise şöyle:
```c++
for(auto canavar: canavarlar) {
	canavarlara_uzaklık[canavar.y][canavar.x] = 0;
    kuyruk.push(canavar);
}
//
while (not kuyruk.empty()) {
   // tahta üzerindeki her noktanın bütün canavarlara uzaklığını bul
   // ...
}
```
Yani ikisi de her noktanın en yakın canavara uzaklığını buluyor. Sonrası temelde aynı. İkisi de şunu yapıyor:
```
c++
kuyru.push(A);
uzaklık[A.y][A.x] = 0;
while (not kuyruk.empty()) {
   // B, yani çıkış noktasını bul
   // Ama A noktasına canavarlardan daha yakın olması gerek
   // ...
}
```
Yani iki program arasındaki hız farkı ilk kısımdan, noktaların canavarlara uzaklığını buluş algoritmasından kaynaklanıyor. Canavar sayısı `c` olsun. Tahtanın büyüklüğü `n x m`. Yavaş programın çalışma hızı `c x n x m`, hızlı olan ise `c + (n x m)` ile doğru orantılı. Neden? Umarım hepiniz bunun neden böyle olduğunu görmüştür. Emin değilseniz bana WhatsApp'ta özelden yazabilirsiniz.
