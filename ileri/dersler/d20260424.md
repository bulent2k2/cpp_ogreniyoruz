3. Bir de Eren arkadaşınız güzel bir sınav sorusuyla uğraşmış (soru 49). Sağolsun bizimle paylaştı. Onun üzerinden geçtik. Değişkenlerin kapsamı (yerel ya da küresel) ve `int a = b = c;` kalıbı hakkında bilgimizi ve dikkatimizi sınıyor:
```c
#include <stdio.h>
int a=3, hak=10;

int f(int a) {
    a -= 8;
    return (hak*a);
}
int g(int hak) {
    return (hak*a);
}
int main() {
    int a=hak=5;
    a = a + f(hak) + 4;
    hak = g(a);
    printf("hak = %d\n", hak++);
}
```
Güzel bir soru ama çok kötü yazılmış bir kod. Tabii kasıtlı. Yerel ve küresel değişkenleri karmakarışık adlandırarak bizi dikkat etmeye zorluyor. Gelin **C++**'a tercüme edelim ve değişkenlerin adlarını düzeltelim:
```c++
#include <iostream>
int a1=3, hak1=10;
int f(int a2) {
    a2 -= 8;
    return hak1*a2;
}
int g(int hak2) {
    return hak2*a1;
}
int main() {
    int a3=hak1=5;
    a3 += f(hak1) + 4;
    hak1 = g(a3);
    std::cout << "hak = " << hak++ << std::endl;
}
```
`hak1` yerel bir değişken gibi görünüyor çünkü şu kalıba benziyor:
```c++
int a3, hak3; // yerel değişken
a3 = hak3 = 5;
```
Ama bu farklı elbet. Doğrusu:
```c++
int a3 = 5;
hak1 = 5; // küresel değişken
```
