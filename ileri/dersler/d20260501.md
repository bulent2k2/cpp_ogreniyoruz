Elif arkadaşınız 47. soruyu paylaştı. Soru şu:
```c
include <stdio.h> // bu satır yok
int main() {
    int ctrl, i = 1, j = 1, k = 1;
    while(j++, i=1, k++) {
        if (abs(k) % (int) 1e8 == 0) printf("%d\n", k);
        if (!k) {
            break;
            ctrl = 0;
        } else if (!i) ctrl = 1;
        else ctrl = 2;
    }
    printf("ctrl=%d", ctrl);
}
```
Yukarıdaki program çalıştığında ekrana ne yazar diye sormuşlar. Seçenekler:
```
a) 0
b) 1
c) 2
d) sonsuz döngüye girer 
e) derleme hatası verir
```

Önce döngü içindeki `a, b, c` yapısını yani virgül işlemcisini daha iyi anlayalım diye şu kodu yazdık: 
```c
#include <stdio.h>
int f() { return 5; }
int main() {
    int y=0; // komut, değeri ya da çıktısı yok, (statement)
    int x = (1+1, f(), "a", 3+3, 'k', f()); // deyiş ya da terim (expression)
    x++, y = x++;
    printf("x:%d y:%d", x, y);
    return 0;
}
```
Virgül işlemcisinin temel özellikleri:
1. Soldan sağa değerlendirme garantisi var — soldaki ifadenin yan etkileri, bir sonraki ifade başlamadan tamamlanır.
2. Sonuç, en sağdaki terimin değeri ve türüdür.
3. C'deki tüm işlemciler arasında en düşük önceliğe sahiptir; bu yüzden çoğu zaman parantez gerekir.

Derleme hatası verir yanıtı neden yanlış? 
`stdio.h` başlığı eksik. Ama soru bu program çalıştığında diyor, sanırım onun için. Döngü neden sonsuz değil? Çünkü `k++` uzun bir süre sonra `INT_MAX` sınırına gelir, bir artınca en son bit 1 olur yani sayı eksiye döner. Bir süre sonra da `k == 0` doğru olur. Şurada [onlinegdb](https://onlinegdb.com/IDtEQgpsa) ile çalıştırıp bunu görebilirsiniz. Bir satır ekledim ki `k`'nin nasıl dolaşıp sıfıra döndüğünü görelim. 

Dinamik Program kanıtlarına devam
===
Geçen derste bozuk paralarla verilen bir toplamı bulma sorusuna bakmıştık en son (DP 2). `x` toplam olsun. `f(x)` bu toplama varan en kısa dizinin boyu olsun. Örneğin bozuk paralar `{1, 5, 7}` kuruş olsun. `x=11` için `f(x)=3` olduğunu bulmuştuk. Çünkü en iyi çözüm `11 = 5+5+1`. Sonra da `n` tane bozuk para için şunu kanıtlamıştık: 
```
f(x) = 1 + min{f(x-b1), f(x-b2), ..., f(x-bn)}
```

Bu derste benzer iki soru için kanıt yaptık: 

DP 3: Toplamı verilen bütün dizileri sayalım 
---
Örnek: `{2, 3, 5}` kuruş paraların kümesi olsun. `f(x)`, `x` toplamına sahip dizilerin sayısı olsun. `f(9) = 8` çünkü:
```
2+2+5
2+5+2
5+2+2
2+2+2+3
2+2+3+2
2+3+2+2
3+2+2+2
3+3+3
```

Kanıtı da bir önceki soruya çok benziyor:

```
f(x) = f(x-b1) + f(x-b2) + ... + f(x-bn)
```

Neden? Çünkü toplamı `x-b1` olan bütün dizilerin sayısı `f(x-b1)` ve o dizilere 
`b1` kuruş ekleyince aradığımız toplamı buluyoruz. Sağ taraftaki diğer terimlerin hepsi farklı dizileri sayıyor (neden?) ve bunların sayamadığı başka da bir dizi olamaz (neden?).

Eksi girdiler için fonksiyonun değeri hep sıfır olmalı elbet. Başlangıç değerimiz de `f(0) = 0` çünkü boş dizinin toplamı sıfır diye düşünebiliriz.

Bütün dinamik problemlerde olduğu gibi, bunda da soruyu (denklemin sol tarafı) daha küçük sorulara (denklemin sağ tarafı) indirgedik. 

DP 4: toplamı verilen bütün kümeleri sayalım 
---

Aynı geçen soru gibi ama farklı dizileri değil sadece farklı kümeleri saymak istiyoruz. Bozuk para kümemiz aynı olsun.  O zaman, `f(9) = 3` çünkü, 
```
{2, 5} -> 2+2+5
{2, 3} -> 2+2+2+3
{3} -> 3+3+3
```

Bellman-Ford ve Floyd-Warshall algoritmalarını kanıtlarken yaptığımız gibi, iki boyutlu bir fonksiyon tanımlamak faydalı oluyor. Aradığımız değeri şöyle de yazabiliriz:

```
f(x, {b1, b2, ..., bn})
``` 

Bu fonksiyonla şunu ifade ediyoruz: `x` toplamına sahip ama sadece ikinci girdideki kümenin içindeki bozuk paraları kullanan dizilerin sayısı. Örnek soru için aradığımız yanıt şu: `f(x, {2, 3, 5})`

Bunu şöyle iki küçük probleme bölelim: 5 kuruş kullanan çözümler ve kullanmayan çözümler. Nasıl yazarız? Kullanmayanlar kolay: `f(x, {2, 3})`. Ya kullananlar? Bunun da püf noktası şu: 5 kuruşu en az bir kere kullanmak şart. Onun için `x-5` girdisi önemli. Ama birden fazla da olabileceği için, istediğimiz ifade şu: `f(x - 5, {2, 3, 5})`.

Yani, bir araya getirirsek:
```
f(x, {2,3,5}) = f(x, {2,3}) + f(x-5, {2,3,5})
```
Geçen soruda olduğu gibi, sağdaki iki terimin aynı çözümleri iki kere sayamadığını, ve bütün çözümleri saydığını söyleyebiliriz, değil mi?

Bozuk para alt kümelerini böyle uzun uzun yazmak biraz yoruyor ve çok yer kaplıyor. Onun için, aynı Floyd-Warshall kanıtında olduğu gibi, tek bir sayıyla, `k` ile bütün alt kümeleri şöyle ifade edebiliyoruz: 
``` 
{b1, b2, ..., b(k-1), bk}
1 <= k <= n
```
Yani, birinci bozuk paradan 'k' konumundaki bozuk paraya kadar olanların alt kümesi. Örneğimiz için, 
```
f(x, 1): sadece 2 kuruş parayı kullanırsak kaç çözüm var,
f(x, 2): 2 ve 3 kuruş paraları kullanırsak kaç çözüm var,
f(x, 3) = f(x) yani hepsini kullanabiliriz
```

Daha önce şöyle yazmıştık.
```math
f(x, {2, 3, 5}) = f(x, {2, 3}) + f(x - 5, {2, 3, 5})
```
Şimdi aynı eşitliği daha kısa ifade edebiliriz:
```math
f(x, 3) = f(x, 2) + f(x-5, 3)
```

Sayılara verdiğimiz anlam ne kadar farklı ve faydalı olabiliyor, değil mi?

Artık genel çözümü şöyle yazabiliriz: 
```math
f(x, k) = f(x, k-1) + f(x-bk, k)
```

Ama, bunun şu aşağıdaki eşitliğe denk olduğunu iyi bilmezsek neye yarar? 
```math
f(x, {b1, b2, .., bk-1, bk}) = f(x, {b1, b2, .., bk-1}) + f(x-bk, {b1, b2, .., bk-1, bk})
```

Kanıtı iyice özümsemek önemli. Arkasından yazılımcık kolay olacak. İnşallah gelecek derse. Ama arada siz de bir deneyin, olur mu?
